# core - 基础设施层

本目录负责**核心基础设施组件的初始化和配置**，包括外部服务的适配器封装。

## 子目录

| 目录 | 说明 |
|------|------|
| `adapter/` | 外部服务适配器 — 封装对第三方服务/系统的调用 |

## 适合放置的内容

- **组件初始化**: 数据库连接、缓存客户端、消息队列等
- **适配器接口**: 对外部系统调用的抽象和封装
- **配置加载**: 应用级别的配置读取与解析逻辑

## 设计原则

- 适配器应定义 **interface**，便于测试时 mock
- 基础设施组件应在服务启动时初始化，通过依赖注入传递给 `service` 层
- 避免在此处编写业务逻辑
