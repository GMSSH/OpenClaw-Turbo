# pkg - 可复用工具库

本目录存放**可在多个项目间复用的公共工具包**。与 `internal/` 不同，`pkg/` 下的代码可以被外部模块导入。

## 工具包清单

| 包名 | 说明 |
|------|------|
| `files/` | 文件系统操作工具集 (读写、复制、JSON 序列化) |
| `rpcutil/` | RPC 请求解析与国际化设置工具 |

## 何时使用 `pkg/` vs `internal/`

| 场景 | 放置位置 |
|------|----------|
| 通用工具函数，可跨项目复用 | `pkg/` |
| 仅当前项目使用的工具函数 | `internal/common/` |
| 业务逻辑代码 | `internal/service/` |

## 添加新工具包

1. 在 `pkg/` 下创建新目录，如 `pkg/httputil/`
2. 确保包的功能通用、独立，不依赖业务逻辑
3. 函数和类型需要添加完整的 GoDoc 注释
4. 为工具包编写单元测试
